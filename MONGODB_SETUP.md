# MongoDB Atlas Setup Guide - FREE Permanent Storage

## Why MongoDB Atlas?
- ✅ **FREE 512MB storage** (no credit card required)
- ✅ Permanent cloud storage (unlike Render's ephemeral storage)
- ✅ Automatic backups and replication
- ✅ Global availability
- ✅ Easy Python integration with PyMongo

---

## Step-by-Step Setup (5 minutes)

### 1. Create Free MongoDB Atlas Account

1. Go to: https://www.mongodb.com/cloud/atlas/register
2. Fill in:
   - Email address
   - Password
   - Click "Create your Atlas account"
3. **NO CREDIT CARD NEEDED** ✅

---

### 2. Create Free Cluster

1. After login, click "**Build a Database**"
2. Choose "**M0 FREE**" tier:
   - 512 MB Storage
   - Shared RAM
   - Free Forever
3. Select Provider: **AWS** (recommended)
4. Select Region: Choose closest to you (e.g., `us-east-1`)
5. Cluster Name: `campus-guardian` (or any name)
6. Click "**Create Cluster**" (takes 1-3 minutes)

---

### 3. Create Database User

1. On the "Security Quickstart" page:
   - **Username**: `drone_admin` (or your choice)
   - **Password**: Click "Autogenerate Secure Password" OR create your own
   - **COPY AND SAVE THIS PASSWORD!** ⚠️
2. Click "Create User"

---

### 4. Configure Network Access

1. Click "**Add My Current IP Address**"
   - OR for Render deployment, add: `0.0.0.0/0` (allows all IPs)
2. Click "Finish and Close"

---

### 5. Get Connection String

1. Click "**Connect**" on your cluster
2. Choose "**Connect your application**"
3. Select:
   - Driver: **Python**
   - Version: **3.12 or later**
4. Copy the connection string (looks like):
   ```
   mongodb+srv://drone_admin:<password>@campus-guardian.xxxxx.mongodb.net/?retryWrites=true&w=majority
   ```
5. **Replace `<password>` with the password you saved in Step 3!**

## Troubleshooting

### Connection Failed?

1. **Check password**: Make sure you replaced `<password>` in connection string
2. **Check IP whitelist**: Add `0.0.0.0/0` to Network Access in MongoDB Atlas
3. **Check internet**: Run `ping mongodb.com`

### "Authentication failed"?

- Password is wrong - reset it in MongoDB Atlas:
  - Database Access → Edit user → Reset password

### Still not working?

Run this test:
```bash
python -c "
from pymongo import MongoClient
import os
uri = os.getenv('MONGODB_URI')
print(f'Testing: {uri[:30]}...')
client = MongoClient(uri, serverSelectionTimeoutMS=5000)
print(client.server_info())
print('✅ SUCCESS!')
"
```

3. ✅ Never commit connection string to GitHub
4. ✅ Use environment variables only
5. ✅ Enable MongoDB's built-in encryption
